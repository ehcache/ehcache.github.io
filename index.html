Java's Most Widely-Used Cache

Ehcache is an open source, standards-based cache for boosting performance, offloading your database, and simplifying scalability. It's the most widely-used Java-based cache proven to be robust and full-featured. Ehcache scales from in-process, with one or more nodes, all the way to mixed in-process/out-of-process configurations with terabyte-sized caches.

Ehcache v3: API reboot & javax.cache driven

Ehcache is one of the most feature rich caching APIs out there. It's been grown organically, founded on very early APIs as designed pre-Java5 and we've learned a lot since the early days and made some compromises along the way. But "fixing" these compromises isn't an easy task, often even impossible without breaking backward compatibility. While some caching solutions took very different approaches on it all, the expert group on JSR-107, Terracotta included, put great efforts in trying to come up with a standard API. We believe that the new major version of the API should be based on the standardized API and JSR-107 serves as the basis for it while Ehcache 3.0 extends the specification in many aspects.

Getting started

You can introduce caching to your Java application quite easily with Ehcache. Either by using its new poweful API or the javax.cache API as defined in the JSR-107 specification.

Using the core Ehcache v3 API

While the new Ehcache v3 is still in heavy developement, you can give it a test ride today! We have been working hard to provide all the features from the previous releases to the new version. The core API is stable and should give you a feel for what's coming...

Downloading the jars

You can download the jar directly from GitHub or getting it from the Maven Central Repository: 
    <groupId>org.ehcache</groupId>
    <artifactId>ehcache</artifactId>
    <version>3.0.0.Alpha</version>
When published, the Ehcache 3.0 beta code will be available at GitHub or at the Maven Central Repository using: 
    <groupId>org.ehcache</groupId>
    <artifactId>ehcache</artifactId>
    <version>3.0.0.Beta</version>

Snapshots

For those wanting the most current code, snapshots are published in the Sonatype Snapshots repository. To use the Ehcache 3.0 snapshot from this repository, access to the repository must be added to your Maven settings.xml and the org.ehcache:ehcache version should be changed to 3.0.0-SNAPSHOT.

Add the following to the <repositories> section of settings.xml: 

    <repository>
      <id>ehcache3-snapshots</id>
      <url>https://oss.sonatype.org/content/repositories/snapshots</url>
      <releases><enabled>false</enabled></releases>
      <snapshots><enabled>true</enabled></snapshots>
    </repository>

In your application's POM, the org.ehcache:ehcache dependency shown above is changed to 

    <groupId>org.ehcache</groupId>
    <artifactId>ehcache</artifactId>
    <version>3.0.0-SNAPSHOT</version>
to enable use of Ehcache 3.0 shapshots.

Dependencies

Ehcache 3.0 depends on Simple Logging Facade for Java (SLF4J) to provide logging support. This dependency is available from Maven central using: 
    <groupId>org.slf4j</groupId>
    <artifactId>slf4j-api</artifactId>
    <version>1.7.9</version>
